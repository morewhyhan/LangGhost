# LangGhost — 产品需求文档

## 产品是什么

一个 Obsidian 插件。用户随意用英语写东西，写完一句，AI 自动标出错误，点一下就能改。每个修正附带一句解释。

## 目标用户

英语非母语者，想通过写日记练习英语，语法经常出错，在用 Obsidian。

## 核心体验

**写，标，改。没有别的。**

用户打开 Obsidian 写东西。每写完一句话，错误的词下面自动出现波浪线。红色是语法错了，蓝色是拼错了，黄色是可以写得更地道，绿色是建议把中文换成英语。

右侧侧边栏实时列出当前文件的所有错误，用户可以直接看列表自己手动改，也可以悬停波浪线弹出气泡一键改。两种方式，一个目的。

句子写对了什么都不会发生。安静就是好消息。

就这么简单。

## 用户流程

### 第一次使用

1. 安装插件
2. 设置页填入 API Key 和 API 地址
3. 开始写

### 日常使用

1. 打开 Obsidian，随便写，中英混搭也行
2. 每写完一句（句号、问号、感叹号），AI 自动检查这句话
3. 有错误：波浪线出现，右侧侧边栏同步列出所有错误。没错误：什么都不会发生
4. 看到错误，两种改法：
   - 悬停波浪线，弹出气泡，点"应用"或"忽略"
   - 看侧边栏列表，自己手动改，或点列表里的"应用"按钮
5. 改完，关掉

## 功能要求

### 自动检查

- 用户写完一句话（敲了句号、问号、感叹号），AI 自动检查这句话
- "一句话"的定义：从上一个句末标点（或文件开头、空行之后）到当前句末标点之间的文字，跨行也算同一句
- 如果用户一直用逗号写了一长段没有句号，不触发检查。等用户写了句号再检查
- 如果用户写了一句没有句末标点就停了，不触发检查。用户后续补上标点后触发
- 检查进行中，底部状态栏显示"检查中..."
- 如果网络出错或 Key 无效，状态栏提示错误原因，不弹窗，不打断
- 可以通过命令面板随时开关自动检查

### 波浪线

- 红色：语法错误（时态、语态、句式等）
- 蓝色：拼写错误
- 黄色：表达建议（没错，但有更地道的说法）
- 绿色：中文翻译建议（用户写了中文，AI 建议换成英语）
- 用户手动改了那个词，波浪线自动消失
- 关掉 Obsidian 再打开，波浪线还在。但波浪线关联的是当时的文字内容，如果那段文字已经变了，对应的波浪线消失

### 气泡

悬停波浪线时弹出，包含：

- 错误 → 修正
- 解释，15 个汉字以内，只说规则本身。比如"过去时"而不是"你在描述已经发生的事情所以应该用过去时"
- 如果有多种改法，都列出来让用户选
- 两个按钮：[应用] [忽略]

### 修正行为

- 点"应用"：替换文字，波浪线消失
- 一句话有多处错误时，应用一处修正后，其余波浪线暂时消失，自动重新检查这句话，重新标注剩余错误
- 点"忽略"：波浪线消失，只在当前编辑会话有效。关掉文件再打开，同样的错误会重新标出来
- 用户手动改了波浪线区域的文字（不管是改对了还是改错了），也算一次修正，记入错题本

### 设置页

- API Key 输入
- API 地址（支持自定义，兼容各种接口）
- 错题本文件路径（默认 `LangGhost/errors.md`）
- 一行预估费用提示

### 侧边栏

默认在右侧显示一个错误列表面板。用户也可以拖到左侧。Obsidian 原生支持侧边栏拖放和调整宽度。

面板内容：

- 每个错误显示为一行：**错误词 → 修正**，下方跟类型标签和解释
  - 例：`apple → an apple` `语法` 可数名词需冠词
- 按在文档中出现的位置从上到下排列
- 没有错误时显示「没有发现错误」

交互：

- 点击一条错误，编辑器光标跳转到对应位置
- 每条错误带 [应用] [忽略] 按钮，功能和气泡里的一样
- 应用或忽略后，条目从列表消失
- 新的错误出现时，自动追加到列表

## 中英混写的处理

用户可能写着写着冒出中文。AI 要理解完整意思，把中文部分也翻译成英语作为建议，标绿色波浪线（和黄色的表达建议区分开）。用户可以选择接受翻译，也可以忽略继续用中文。

## 错题记录

- 每次修正一个错误（点"应用"或手动改了波浪线区域的文字），这条错误自动追加到错题本
- 错题本是 vault 里的一个 Markdown 文件，路径在设置页配置
- 记录内容：日期、原文、修正、错误类型、解释
- 不做统计、不排排行、不打分。用户自己翻着看，自然知道哪些地方反复出错

错题本格式示例：

```markdown
## 2026-05-05

- **go → went** `语法` 过去时，描述已发生的事
- **buyed → bought** `语法` 不规则动词过去式
- **delicous → delicious** `拼写` 中间有两个 i

## 2026-05-06

- **she don't → she doesn't** `语法` 第三人称单数用 doesn't
```

## 不做什么

- 不做趋势图、打卡
- 不做词汇本
- 不做学习报告
- 不做移动端适配
- 不做永久忽略列表
- 不做英语水平评估或打分
